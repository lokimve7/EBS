# Task 01-11 - 이미지 탭 씬별 OpenAI 이미지 생성

## 설명
이미지 탭의 각 씬 카드에서 `✦ 이미지 생성` 버튼을 눌렀을 때 해당 장면의 이미지 생성용 프롬프트를 사용해 OpenAI API에 이미지를 요청할 수 있어야 한다. 생성이 진행되는 동안에는 요청한 카드만 로딩 상태로 보여야 하고, 생성이 끝나면 해당 카드의 이미지 영역에 결과 이미지가 표시되어야 한다. 또한 생성이 완료된 이미지는 현재 프로젝트 폴더 안의 파일로 저장되어 이후 다시 활용할 수 있어야 한다.

## 구현 항목
- [x] 이미지 탭의 각 씬 카드에서 `✦ 이미지 생성` 버튼을 누르면 해당 장면의 이미지 생성용 프롬프트를 기준으로 OpenAI API에 이미지 생성을 요청한다.
- [x] 이미지 생성 요청이 진행되는 동안에는 요청한 씬 카드만 로딩 상태로 표시된다.
- [x] 이미지 생성이 완료되면 요청한 씬 카드의 이미지 영역에 결과 이미지가 표시된다.
- [x] 생성이 완료된 이미지는 현재 프로젝트 폴더 안의 파일로 저장된다.

## 범위 밖
- 여러 씬 이미지를 한 번에 생성하는 일괄 생성 기능
- 이미지 생성 프롬프트를 화면에서 직접 편집하는 기능
- `↻`, `↓` 버튼의 실제 동작 연결
- 생성된 이미지에 대한 추가 편집, 변형, 업스케일 기능

## 사전 전제
- `Tasks/01-6.md`의 장면 스크립트 생성 결과에 장면별 이미지 생성용 프롬프트가 포함되어 있어야 한다.
- `Tasks/01-8.md`의 프로젝트별 저장 폴더 구조가 준비되어 있어야 한다.
- `Tasks/01-10.md`의 이미지 탭 씬 카드와 `✦ 이미지 생성` 버튼 구성이 준비되어 있어야 한다.

## 수동 작업 (구현 후 구체화)
- 상단 `API 키 설정` 창을 열고 `OpenAI API Key` 입력칸에 실제 사용할 OpenAI API 키를 입력한 뒤 `저장`을 누른다.

- [x] `이미지` 탭의 한 씬 카드에서 `✦ 이미지 생성` 버튼을 눌렀을 때 해당 카드 버튼이 `생성 중...`으로 바뀌고, 같은 카드 이미지 영역에 로딩 표시가 보이는지 확인한다.
- [x] 이미지 생성이 끝나면 방금 요청한 카드의 이미지 영역에 실제 생성된 이미지가 보이는지 확인한다.
- [x] 다른 씬 카드는 동시에 로딩 상태로 바뀌지 않고 기존 상태를 유지하는지 확인한다.
- [x] 생성이 완료된 카드 하단에 이미지 저장 경로가 표시되는지 확인한다.
- [x] 프로젝트 폴더의 `images/` 안에 해당 씬 번호로 저장된 이미지 파일이 실제로 생성되었는지 확인한다.
- [x] OpenAI API 키가 없거나 잘못된 상태에서 `✦ 이미지 생성` 버튼을 눌렀을 때 오류 또는 안내 상태를 통해 실패 원인을 구분할 수 있는지 확인한다.
