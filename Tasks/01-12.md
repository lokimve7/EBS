# Task 01-12 - 프로젝트 대표 JSON 기반 통합 복원

## 설명
상단의 `프로젝트 불러오기` 흐름에서 프로젝트를 대표하는 JSON 파일 하나를 선택했을 때, 저장 당시의 프로젝트 상태가 화면에 함께 복원되어야 한다. 선택한 파일을 기준으로 `스크립트` 탭의 입력값과 씬 카드 목록뿐 아니라 `이미지` 탭의 씬 카드와 생성된 이미지 상태도 같이 이어져야 하며, 사용자는 탭을 다시 오가더라도 같은 프로젝트 상태를 계속 확인할 수 있어야 한다.

## 구현 항목
- [x] 프로젝트를 대표하는 JSON 파일을 선택해 불러오는 흐름이 제공된다.
- [x] 불러온 프로젝트 데이터로 `스크립트` 탭의 입력값과 씬 카드 목록이 함께 복원된다.
- [x] 불러온 프로젝트 데이터로 `이미지` 탭의 씬 카드 목록과 생성된 이미지 상태가 함께 복원된다.
- [x] 불러온 파일이 현재 프로젝트 형식과 맞지 않거나 복원에 필요한 데이터가 부족할 때 사용자에게 실패 상태를 구분해서 보여준다.

## 범위 밖
- 프로젝트 목록을 화면 안에서 탐색하거나 최근 프로젝트 목록을 별도로 관리하는 기능
- 불러온 프로젝트를 수정한 뒤 다시 저장하는 편집 기능
- 영상, 보이스 등 다른 결과물 탭까지 함께 복원하는 기능
- 여러 프로젝트를 한 번에 비교하거나 병합하는 기능

## 사전 전제
- `Tasks/01-8.md`의 프로젝트별 저장 폴더 구조가 준비되어 있어야 한다.
- `Tasks/01-9.md`의 프로젝트 파일 선택 및 스크립트 탭 복원 흐름이 준비되어 있어야 한다.
- `Tasks/01-11.md`의 이미지 생성 결과 저장 흐름이 준비되어 있어야 한다.

## 수동 작업 (구현 후 구체화)
- 먼저 이미지 생성까지 완료된 프로젝트가 하나 있어야 하므로, `스크립트` 탭에서 장면을 생성한 뒤 `이미지` 탭에서 최소 1개 이상의 씬 이미지를 생성해 둔다.
- 생성이 끝난 프로젝트 폴더 안의 `project.json` 파일 위치를 미리 확인해 둔다.

### 결과 확인 (구현 후 구체화)
- [x] 터미널에서 서버를 실행한 뒤 브라우저에서 `http://localhost:3001`을 열고, 상단 `프로젝트 불러오기` 버튼으로 저장된 프로젝트의 `project.json` 파일을 선택한다.
- [x] 파일을 불러온 직후 `스크립트` 탭의 `영상 주제`, `분위기`, `스타일`, `장면 수` 입력값과 씬 카드 목록이 저장 당시 내용으로 복원되는지 확인한다.
- [x] `이미지` 탭으로 이동했을 때 저장 당시 이미지가 생성된 씬 카드에는 생성된 이미지가 바로 보이고, 아직 생성하지 않은 씬 카드는 빈 이미지 상태로 유지되는지 확인한다.
- [x] 이미지가 복원된 씬 카드 하단에 저장 경로가 함께 표시되는지 확인한다.
- [x] 불러온 뒤 `스크립트` 탭과 `이미지` 탭을 다시 오가도 같은 프로젝트 상태가 계속 유지되는지 확인한다.
- [x] 장면 데이터가 없거나 형식이 맞지 않는 JSON 파일을 선택했을 때 상태 배너에 실패 안내가 보이는지 확인한다.
