# Task 01-13 - 프로젝트 대표 상태 파일 분리

## 설명
프로젝트를 다시 불러오기 위한 대표 상태 파일은 `project.json`으로 분리하고, `scripts/scene-script.json`은 장면 스크립트 전용 파일로 유지되어야 한다. `project.json`에는 실제 스크립트 본문을 중복 저장하지 않고 어떤 스크립트 파일과 어떤 이미지 파일을 현재 프로젝트가 사용하는지만 기록해야 하며, 사용자는 `project.json` 하나를 선택해 기존과 같은 복원 흐름을 계속 사용할 수 있어야 한다.

## 구현 항목
- [x] 프로젝트 저장 시 `project.json` 대표 상태 파일이 함께 생성된다.
- [x] `scripts/scene-script.json`에는 스크립트 전용 데이터만 저장되고 이미지 복원 상태는 포함되지 않는다.
- [x] `project.json`에는 스크립트 본문을 중복 저장하지 않고 연결된 스크립트 파일과 이미지 파일 참조만 저장된다.
- [x] 프로젝트 불러오기는 `project.json` 기준으로 연결된 스크립트 파일과 이미지 파일을 다시 읽어 스크립트 탭과 이미지 탭 상태를 함께 복원한다.
- [x] 기존 `scripts/scene-script.json`만 남아 있는 프로젝트 파일을 선택했을 때도 가능한 범위에서 호환 불러오기가 유지되거나 실패 사유가 구분되어 안내된다.

## 범위 밖
- 기존 저장 프로젝트 전체를 한 번에 일괄 마이그레이션하는 기능
- 프로젝트 목록 탐색 화면 추가
- 영상, 보이스 상태까지 저장 구조를 확장하는 작업
- 프로젝트 편집 후 수동 저장 기능

## 사전 전제
- `Tasks/01-8.md`의 프로젝트별 저장 폴더 구조가 준비되어 있어야 한다.
- `Tasks/01-11.md`의 이미지 생성 및 저장 흐름이 준비되어 있어야 한다.
