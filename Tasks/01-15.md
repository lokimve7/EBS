# Task 01-15 - 비디오 탭 씬별 영상 생성

## 설명
비디오 탭의 각 씬 카드에서 `✦ 영상 생성` 버튼을 눌렀을 때, 해당 장면에 연결된 이미지와 영상 생성 프롬프트를 사용해 EvoLink.AI의 Kling 영상 생성을 요청할 수 있어야 한다. 생성이 진행되는 동안에는 해당 카드만 로딩 상태로 보여야 하고, 생성이 완료되면 같은 카드의 영상 영역에서 결과 영상을 확인할 수 있어야 한다. 또한 완료된 영상은 현재 프로젝트 폴더의 `videos/` 경로에 저장되고, 이후 `project.json` 기반 프로젝트 불러오기 시 영상 탭 상태도 함께 복원되어야 한다.

## 구현 항목
- [x] 비디오 탭의 각 씬 카드 `✦ 영상 생성` 버튼이 해당 장면 기준의 영상 생성 요청을 시작한다.
- [x] 영상 생성 요청에는 해당 장면의 이미지와 영상 생성 프롬프트가 함께 전달된다.
- [x] 영상 생성이 진행되는 동안 해당 씬 카드에 로딩 상태가 표시된다.
- [x] 영상 생성이 완료되면 해당 씬 카드의 영상 영역에 결과 영상이 표시된다.
- [x] 생성이 완료된 영상은 현재 프로젝트 폴더의 `videos/` 경로 파일로 저장된다.
- [x] `project.json` 기반 프로젝트 불러오기 시 저장된 영상 상태가 비디오 탭 카드에 함께 복원된다.

## 범위 밖
- 영상 다운로드 버튼 동작 구현
- 영상 생성 실패 원인별 상세 분류 또는 재시도 정책 확장
- 다른 영상 모델 또는 다른 외부 서비스 선택 기능 추가

## 사전 전제
- `Tasks/01-11.md`의 이미지 생성 결과가 장면별로 준비되어 있어야 한다.
- `Tasks/01-14.md`의 비디오 탭 씬 카드 레이아웃과 장면 연동 구조가 준비되어 있어야 한다.

## 수동 작업 (구현 후 구체화)
