# Task 01-16 - SQLite 프로젝트 기본 정보 저장 구조

## 설명
프로젝트 정보를 파일 중심 구조에서 DB 기반 구조로 확장하기 위한 첫 단계로, 단일 서버에서 운영하기 쉬운 SQLite 기반 저장 구조를 준비한다. 서버 시작 시 DB 파일과 필요한 테이블이 없으면 자동으로 생성되어야 하며, 프로젝트 목록, 프로젝트 기본 정보, 생성일, 수정일을 DB에 저장하고 불러올 수 있는 기본 구조가 마련되어야 한다. 기존 `projects/` 폴더 저장 구조는 이번 단계에서 제거하지 않고 유지한다.

## 구현 항목
- [x] 서버 시작 시 SQLite DB 파일이 없으면 자동으로 생성된다.
- [x] 서버 시작 시 프로젝트 기본 정보 저장에 필요한 테이블이 없으면 자동으로 생성된다.
- [x] 프로젝트 목록을 DB에 저장하고 불러올 수 있는 기본 구조가 준비된다.
- [x] 프로젝트 기본 정보, 생성일, 수정일을 DB에 저장할 수 있다.
- [x] 기존 `projects/` 폴더 기반 저장 구조가 계속 유지된다.

## 범위 밖
- 기존 `projects/` 폴더 데이터 전체 마이그레이션
- 스크립트 장면 데이터의 DB JSON 저장
- 이미지와 영상 결과 상태의 DB 저장
- 파일 선택 방식의 프로젝트 불러오기를 DB 목록 선택 방식으로 변경
- 여러 사용자 동시 접속 저장 안정화

## 사전 전제
- `Tasks/01-13.md`의 `project.json` 기반 프로젝트 대표 상태 파일 구조가 준비되어 있어야 한다.
- `Tasks/01-15.md`의 이미지와 영상 결과물이 기존 `projects/` 폴더 구조에 저장되고 복원되는 흐름이 유지되어야 한다.
