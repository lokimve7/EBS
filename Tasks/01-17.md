# Task 01-17 - 이미지/영상 결과 URL DB 저장

## 설명
이미지와 영상 결과 파일 자체는 기존처럼 내 PC의 로컬 프로젝트 폴더에 저장하되, DB에는 브라우저에서 해당 파일을 볼 수 있는 URL과 생성 상태 정보를 저장한다. 이미지 또는 영상 생성이 완료되면 씬 번호, 결과 타입, URL, 생성 상태, 오류 메시지, 프롬프트 같은 결과 메타데이터가 DB에 남아야 하며, 프론트엔드는 저장된 파일 경로를 직접 추측하지 않고 DB에서 받은 URL을 기준으로 이미지와 영상을 표시할 수 있어야 한다.

## 구현 항목
- [x] 이미지 생성이 완료되면 이미지 파일은 로컬 프로젝트 폴더에 저장되고, DB에는 해당 이미지를 볼 수 있는 URL이 저장된다.
- [x] 영상 생성이 완료되면 영상 파일은 로컬 프로젝트 폴더에 저장되고, DB에는 해당 영상을 볼 수 있는 URL이 저장된다.
- [x] DB에는 씬 번호, 결과 타입, URL, 생성 상태, 오류 메시지, 프롬프트 정보가 저장된다.
- [x] 이미지와 영상 생성 실패 상태도 DB에 생성 상태와 오류 메시지로 저장된다.
- [x] 프론트엔드는 DB에서 받은 URL을 사용해 이미지와 영상을 표시한다.
- [x] 기존 `projects/` 폴더 기반 파일 저장 구조는 계속 유지된다.

## 범위 밖
- 스크립트 장면 데이터를 DB JSON으로 저장하는 구조 추가
- DB 기반 프로젝트 목록 선택 화면 추가
- 기존 `projects/` 폴더 데이터의 DB 마이그레이션
- 여러 사용자 동시 접속 저장 안정화
- 로컬 파일 저장 구조 제거
