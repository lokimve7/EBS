# Task 01-18 - DB 프로젝트 목록 기반 불러오기

## 설명
상단 `프로젝트 불러오기` 흐름을 로컬 파일 선택 방식이 아니라 서버가 제공하는 DB 프로젝트 목록에서 선택하는 방식으로 바꾼다. 같은 네트워크의 다른 PC에서 접속한 사용자도 서버 DB에 저장된 프로젝트 이름, 생성일, 수정일을 보고 프로젝트를 선택할 수 있어야 하며, 선택한 프로젝트의 스크립트 장면과 이미지/영상 URL 상태가 함께 복원되어야 한다.

## 구현 항목
- [x] 상단 `프로젝트 불러오기`를 누르면 로컬 파일 선택 대신 서버에 저장된 프로젝트 목록을 선택할 수 있다.
- [x] 프로젝트 목록에는 프로젝트 이름, 생성일, 수정일이 표시된다.
- [x] 프로젝트를 선택하면 DB에 저장된 프로젝트 정보와 연결된 스크립트 장면 데이터가 복원된다.
- [x] 프로젝트를 선택하면 DB에 저장된 이미지와 영상 URL 상태가 함께 복원된다.
- [x] 같은 네트워크의 다른 PC에서 접속해도 동일한 서버의 프로젝트 목록과 결과물을 볼 수 있다.
- [x] 기존 프로젝트 저장 폴더와 결과물 파일 제공 흐름은 유지된다.
- [x] 다른 PC에서 복원된 영상 파일을 재생할 수 있도록 서버의 영상 파일 응답이 브라우저 Range 요청을 처리한다.

## 범위 밖
- 기존 `projects/` 폴더 데이터 전체 마이그레이션
- 여러 사용자가 동시에 같은 프로젝트를 수정할 때의 충돌 처리
- 사용자 계정, 권한, 로그인 기능
- 로컬 파일 저장 구조 제거
- 스크립트, 이미지, 영상의 새 생성 기능 변경

## 사전 전제
- `Tasks/01-16.md`의 SQLite 프로젝트 기본 정보 저장 구조가 준비되어 있어야 한다.
- `Tasks/01-17.md`의 이미지/영상 결과 URL DB 저장 흐름이 준비되어 있어야 한다.
- 프로젝트 결과물 파일은 서버가 접근 가능한 `projects/` 폴더 안에 유지되어야 한다.

## 수동 작업 (구현 후 구체화)
- 같은 네트워크의 다른 PC에서 접속 확인을 하려면, 서버를 실행하는 PC의 내부 IP 주소를 확인한다.

### 결과 확인 (구현 후 구체화)
- [x] 터미널에서 `npm start`를 실행했을 때 서버가 정상 시작되고 `http://localhost:3001` 접속 안내와 같은 네트워크 접속 안내가 보이는지 확인한다.
- [x] 브라우저에서 `http://localhost:3001`을 열고 상단 `프로젝트 불러오기` 버튼을 눌렀을 때 프로젝트 목록 창이 열리는지 확인한다.
- [x] 프로젝트 목록 창에 프로젝트 이름, 생성일, 수정일이 보이는지 확인한다.
- [x] 프로젝트 하나를 선택했을 때 스크립트 탭의 입력값과 장면 카드가 선택한 프로젝트 내용으로 바뀌는지 확인한다.
- [x] 이미지 탭과 영상 탭에서 선택한 프로젝트의 저장된 이미지와 영상 상태가 카드에 함께 복원되는지 확인한다.
- [x] 같은 네트워크의 다른 PC 브라우저에서 `http://<서버PC내부IP>:3001`로 접속했을 때 같은 프로젝트 목록이 보이고, 선택한 프로젝트의 결과물이 표시되는지 확인한다.
- [x] 같은 네트워크의 다른 PC에서 불러온 프로젝트의 영상 카드가 검은 빈 영역이나 로드 실패 상태가 아니라 재생 가능한 컨트롤로 표시되는지 확인한다.
