# Task 01-19 - 프로젝트 고유 ID 기반 저장 안정화

## 설명
여러 사람이 같은 서버에 접속해서 프로젝트를 만들거나 불러와도 데이터가 섞이지 않도록 프로젝트마다 고유 ID를 사용해 저장 경로와 DB 데이터를 구분한다. 이미지와 영상 결과 파일도 씬 번호만으로 구분하지 않고 프로젝트 ID와 결과 타입을 기준으로 충돌하지 않게 저장되어야 하며, 동시에 여러 요청이 들어와도 서로 다른 프로젝트의 상태가 덮어써지지 않아야 한다.

## 구현 항목
- [x] 새 프로젝트가 만들어질 때 프로젝트마다 고유 ID가 부여된다.
- [x] 프로젝트 저장 경로와 DB 데이터가 프로젝트 고유 ID를 기준으로 구분된다.
- [x] 이미지와 영상 파일명이 씬 번호만으로 만들어지지 않고 프로젝트 ID와 결과 타입을 포함해 충돌하지 않는다.
- [x] 동시에 여러 요청이 들어와도 서로 다른 프로젝트의 스크립트, 이미지, 영상 상태가 서로 덮어써지지 않는다.
- [x] 이미지 또는 영상 생성 오류가 발생하면 어느 프로젝트와 어느 씬에서 실패했는지 화면 또는 저장 데이터에서 확인할 수 있다.
- [x] 기존 프로젝트 목록 불러오기와 스크립트, 이미지, 영상 상태 복원 흐름은 유지된다.

- 같은 프로젝트를 여러 사용자가 동시에 편집할 때의 세부 충돌 해결 UI
- 기존 `projects/` 폴더 전체 데이터 마이그레이션
- 로컬 파일 저장 구조 제거
- 이미지 또는 영상 생성 API 자체의 동작 변경

- 기존 프로젝트 결과물 파일은 서버가 접근 가능한 `projects/` 폴더 안에 유지되어야 한다.
