# Task 01-8 - 프로젝트별 결과 저장 폴더 구조 준비

## 설명
스크립트 생성이 완료되면 프로젝트별로 결과 파일이 한 폴더에 모이도록 저장 구조를 준비해야 한다. 생성된 스크립트 결과는 자동으로 저장되어야 하며, 이후 단계에서 이미지, 영상, 보이스 같은 결과물을 종류별로 나누어 저장할 수 있도록 프로젝트 폴더 아래의 하위 폴더 구조도 함께 준비되어 있어야 한다.

## 구현 항목
- [x] 스크립트 생성이 완료되면 프로젝트별 결과 저장 폴더가 없을 때 자동으로 생성된다.
- [x] 프로젝트별 결과 저장 폴더 아래에는 이후 이미지, 영상, 보이스 등을 따로 저장할 수 있는 하위 폴더 구조가 함께 준비된다.
- [x] 생성된 장면 스크립트 데이터는 해당 프로젝트 폴더 안의 스크립트 저장 위치에 자동으로 저장된다.
- [x] 같은 프로젝트 폴더가 이미 있으면 기존 폴더 구조를 유지한 채 스크립트 결과만 현재 생성 내용으로 갱신된다.

## 범위 밖
- 이미지, 영상, 보이스 생성 기능 자체 구현
- 저장된 프로젝트 목록을 화면에서 조회하거나 선택하는 기능
- 저장된 결과 파일의 다운로드, 삭제, 이름 변경 기능
- 프로젝트 이름 규칙 고도화나 중복 이름 충돌 해결 정책 확장

## 사전 전제
- `Tasks/01-6.md`의 스크립트 생성 완료 흐름이 준비되어 있어야 한다.

## 수동 작업 (구현 후 구체화)
- 상단 `API 키 설정` 창을 열고 `OpenAI API Key` 입력칸에 실제 사용할 OpenAI API 키를 입력한 뒤 `저장`을 누른다.
- `스크립트` 탭의 `영상 주제` 입력칸에 프로젝트 폴더 이름으로도 사용할 주제를 입력한다.
