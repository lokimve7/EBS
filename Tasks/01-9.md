# Task 01-9 - 저장된 프로젝트 불러오기

## 설명
상단의 `프로젝트 불러오기` 버튼으로 저장된 프로젝트 파일을 직접 선택해 다시 불러올 수 있어야 한다. 사용자가 저장된 프로젝트의 스크립트 파일을 선택하면 해당 장면 데이터를 읽어 현재 화면의 `스크립트` 탭에 반영하고, 씬 카드 목록이 저장 당시 내용에 맞게 다시 구성되어야 한다.

## 구현 항목
- [x] 상단 `프로젝트 불러오기` 버튼을 누르면 저장된 프로젝트 파일을 선택할 수 있는 흐름이 제공된다.
- [x] 사용자가 프로젝트의 스크립트 파일을 선택하면 파일 안의 장면 데이터를 읽어 올 수 있다.
- [x] 불러온 장면 데이터로 `스크립트` 탭의 씬 카드 목록이 다시 구성된다.
- [x] 불러온 파일이 현재 프로젝트 형식과 맞지 않을 때 사용자에게 실패 상태를 구분해서 보여준다.

## 범위 밖
- 프로젝트 목록을 화면 안에서 탐색하거나 최근 프로젝트 목록을 별도로 관리하는 기능
- 불러온 프로젝트를 수정한 뒤 다시 저장하는 편집 기능
- 이미지, 영상, 보이스 폴더 안의 결과물을 함께 불러오는 기능
- 여러 프로젝트 파일을 한 번에 비교하거나 병합하는 기능

## 사전 전제
- `Tasks/01-8.md`의 프로젝트별 저장 폴더와 `scripts/scene-script.json` 저장 흐름이 준비되어 있어야 한다.

## 수동 작업 (구현 후 구체화)
- `Tasks/01-8.md` 결과로 저장된 프로젝트 폴더 안의 `scripts/scene-script.json` 파일 위치를 미리 확인해 둔다.

## 완료 조건

### 에이전트 확인
- [x] 관련 코드 수정 완료
- [x] 로컬 정적 점검 또는 프로젝트 구조 기준 확인 완료
- [x] HTML/CSS/JavaScript 규칙 위반 없음
- [x] 현재 task 문서가 실제 구현 기준으로 갱신됨

### 결과 확인 (구현 후 구체화)
- [ ] 터미널에서 서버를 실행한 뒤 브라우저에서 `http://localhost:3001`을 연다.
- [ ] 상단 `프로젝트 불러오기` 버튼을 눌렀을 때 파일 탐색기 창이 열리는지 확인한다.
- [ ] 저장된 프로젝트의 `scripts/scene-script.json` 파일을 선택했을 때 `스크립트` 탭의 씬 카드 목록이 저장된 장면 수와 내용에 맞게 다시 구성되는지 확인한다.
- [ ] 프로젝트 파일을 불러온 뒤 `영상 주제`, `분위기`, `스타일`, `장면 수` 입력값도 저장된 내용에 맞게 바뀌는지 확인한다.
- [ ] 장면 데이터가 없는 JSON 파일 또는 형식이 맞지 않는 JSON 파일을 선택했을 때 상태 배너에 실패 안내가 보이는지 확인한다.
