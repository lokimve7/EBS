# DB 구조 전환 단계별 요청 초안

## 1단계: 서버를 같은 망에서 접속 가능하게 변경

## 원하는 동작

- 내 PC에서 실행 중인 서버를 같은 네트워크에 있는 다른 PC에서도 접속할 수 있게 하고 싶어.

## 세부 설명

- 서버가 `localhost`만 받는 구조가 아니라 같은 LAN의 다른 PC에서도 `내PC_IP:3001` 주소로 접속 가능해야 해.
- 기존 메인 화면과 탭 기능은 그대로 동작해야 해.
- 서버 실행 시 접속 가능한 주소를 콘솔에 안내해줘.
- 방화벽이나 네트워크 설정은 코드에서 자동 처리하지 말고, 필요한 안내만 남겨줘.

## 2단계: DB 기본 구조 추가

## 원하는 동작

- 프로젝트 정보를 파일 중심이 아니라 DB에 저장하고 불러올 수 있는 기본 구조를 만들고 싶어.

## 세부 설명

- 내 PC에서 단일 서버로 운영하기 쉬운 SQLite DB를 사용하고 싶어.
- 프로젝트 목록, 프로젝트 기본 정보, 생성일, 수정일을 DB에 저장해야 해.
- 서버 시작 시 DB 파일과 필요한 테이블이 없으면 자동으로 생성되게 해줘.
- 기존 `projects/` 폴더 저장 구조는 바로 제거하지 말고 유지해줘.

## 3단계: 스크립트 데이터를 DB JSON으로 저장

## 원하는 동작

- 스크립트 생성 결과를 `scene-script.json` 파일뿐 아니라 DB에도 JSON 데이터로 저장하고 싶어.

## 세부 설명

- 스크립트 생성이 완료되면 장면별 스크립트 데이터를 DB에 JSON 형태로 저장해야 해.
- 프로젝트 설정값도 함께 DB에 저장되어야 해.
- 프로젝트를 불러올 때 DB에 저장된 스크립트 JSON을 기준으로 스크립트 탭의 입력값과 씬 카드가 복원되어야 해.
- 기존 파일 기반 불러오기도 당장은 깨지지 않게 유지해줘.

## 4단계: 이미지/영상 결과를 URL 기반으로 DB에 저장

## 원하는 동작

- 이미지와 영상 파일 자체는 로컬 파일로 저장하고, DB에는 해당 파일을 볼 수 있는 URL과 상태값을 저장하고 싶어.

## 세부 설명

- 이미지 생성이 완료되면 이미지 파일은 내 PC 로컬 저장소에 저장하고, DB에는 이미지 URL을 저장해야 해.
- 영상 생성이 완료되면 영상 파일은 내 PC 로컬 저장소에 저장하고, DB에는 영상 URL을 저장해야 해.
- DB에는 씬 번호, 결과 타입, URL, 생성 상태, 오류 메시지, 프롬프트 같은 정보를 저장해야 해.
- 프론트엔드는 DB에서 받은 URL로 이미지와 영상을 표시해야 해.

## 5단계: DB 기반 프로젝트 목록과 불러오기 기능 추가

## 원하는 동작

- 다른 PC에서 접속한 사용자도 DB에 저장된 프로젝트 목록을 보고 선택해서 불러올 수 있게 하고 싶어.

## 세부 설명

- 상단의 `프로젝트 불러오기` 흐름을 파일 선택 방식이 아니라 서버의 프로젝트 목록에서 선택하는 방식으로 바꾸고 싶어.
- 프로젝트 목록에는 프로젝트 이름, 생성일, 수정일이 보여야 해.
- 프로젝트를 선택하면 DB의 스크립트 JSON과 이미지/영상 URL 상태가 함께 복원되어야 해.
- 같은 네트워크의 다른 PC에서 접속해도 동일한 프로젝트 목록과 결과물을 볼 수 있어야 해.

## 6단계: 기존 projects 파일 구조에서 DB 구조로 마이그레이션

## 원하는 동작

- 기존 `projects/` 폴더에 저장된 프로젝트들을 DB 구조로 옮길 수 있게 하고 싶어.

## 세부 설명

- 기존 `projects/<project>/project.json`과 `scripts/scene-script.json`을 읽어서 DB에 등록할 수 있어야 해.
- 기존 이미지와 영상 파일은 삭제하지 말고, 해당 파일을 볼 수 있는 URL만 DB에 저장해야 해.
- 이미 DB에 등록된 프로젝트는 중복 등록되지 않게 처리해야 해.
- 마이그레이션 후에는 DB 기반 프로젝트 불러오기로 기존 결과물을 다시 볼 수 있어야 해.

## 7단계: 동시 접속 저장 안정화

## 원하는 동작

- 여러 사람이 같은 서버에 접속해서 프로젝트를 만들거나 불러와도 데이터가 섞이지 않게 하고 싶어.

## 세부 설명

- 프로젝트마다 고유 ID를 사용해서 저장 경로와 DB 데이터를 구분해야 해.
- 이미지, 영상 파일명도 씬 번호만으로 만들지 말고 프로젝트 ID와 결과 타입을 기준으로 충돌하지 않게 저장해야 해.
- 동시에 여러 요청이 들어와도 서로 다른 프로젝트의 상태가 덮어써지지 않아야 해.
- 오류가 발생하면 어느 프로젝트와 어느 씬에서 실패했는지 알 수 있게 저장하거나 표시해줘.
