# Architecture

## 목적
이 문서는 AI 에이전트가 파일을 만들거나 이동할 때 폴더 책임, 파일 배치, 화면 파일 이름 규칙을 확인하는 기준 문서입니다.

## 구조 원칙

- 프론트엔드(정적 화면) 구조와 백엔드(서버) 책임을 분리합니다.
- 지금 필요한 범위에서만 구조를 확장합니다.

## 공통 폴더 책임

- `docs/`: 하네스 문서와 작업 규칙
- `Tasks/`: 개별 task 문서
- `public/`: 브라우저에 바로 보여줄 정적 화면 파일
- `projects/`: 스크립트 생성 이후 프로젝트별 결과 JSON과 이후 이미지, 영상, 보이스 결과물을 모아 둘 저장 폴더
- `data/`: 단일 서버 실행에 필요한 SQLite DB 파일을 보관하는 폴더

## 현재 프로젝트 폴더 구조와 책임

<!-- 새 프로젝트 시작 시 이 구역의 항목을 초기화한다. -->
- `server.js`: 같은 네트워크에서 접근 가능한 `3001` 포트에서 메인 화면과 정적 CSS, JavaScript 파일, `projects/` 안의 저장 이미지 파일과 Range 요청을 지원하는 영상 파일을 제공하고, OpenAI 기반 스크립트 생성 요청과 씬별 이미지 생성 요청, EvoLink.AI Kling 기반 씬별 영상 생성 요청을 중계하며 프로젝트 고유 ID 기반 결과 저장 폴더, 프로젝트 대표 JSON 저장, DB 프로젝트 기본 정보 저장 호출, 프로젝트 목록 API, 선택 프로젝트 상세 API, 이미지/영상 결과 메타데이터 저장 호출과 결과 목록 API, 프로젝트 ID와 결과 타입을 포함한 이미지/영상 파일 저장과 이미지/영상 상태 반영을 처리하는 Node.js 서버 진입 파일
- `database.js`: `node:sqlite` 기반으로 `data/ebs.sqlite` 파일과 프로젝트 고유 ID를 포함한 프로젝트 기본 정보 테이블, 프로젝트 고유 ID를 포함한 이미지/영상 결과 메타데이터 테이블을 준비하고 프로젝트 기본 정보 저장 및 목록/상세 조회, 프로젝트 수정일 갱신, 씬별 이미지/영상 결과 저장 및 조회 함수를 제공하는 DB 모듈
- `package.json`: 서버 실행 스크립트를 정의하는 프로젝트 설정 파일
- `data/.gitkeep`: SQLite DB 파일 보관 폴더를 저장소에 유지하기 위한 기준 파일
- `data/ebs.sqlite`: 서버 시작 시 자동 생성되며 프로젝트 기본 정보와 이미지/영상 결과 URL, 생성 상태, 오류 메시지, 프롬프트 정보를 보관하는 SQLite DB 파일
- `projects/.gitkeep`: 프로젝트별 결과 저장 루트 폴더를 저장소에 유지하기 위한 기준 파일
- `projects/<project-id>-<project>/project.json`: 현재 프로젝트가 사용하는 프로젝트 고유 ID, 스크립트 파일 경로와 씬별 이미지 파일 경로, 씬별 영상 파일 경로, 공통 프로젝트 설정값을 보관하는 프로젝트 대표 상태 파일
- `projects/<project-id>-<project>/scripts/scene-script.json`: 프로젝트 고유 ID와 장면 스크립트 전용 데이터만 저장하는 스크립트 결과 파일
- `projects/<project-id>-<project>/images/<project-id>-image-scene-XX.png`: 프로젝트 ID, 결과 타입, 장면 번호를 포함해 저장되는 장면별 이미지 결과 파일
- `projects/<project-id>-<project>/videos/<project-id>-video-scene-XX.mp4`: 프로젝트 ID, 결과 타입, 장면 번호를 포함해 저장되는 장면별 영상 결과 파일
- `public/index.html`: 공통 상단 영역, 프로젝트 불러오기 버튼, 탭 메뉴, `스크립트` 탭 화면, 동적 렌더링을 받는 `이미지` 탭과 `영상` 탭의 씬 카드 영역, DB 프로젝트 목록 선택 모달, OpenAI와 EvoLink.AI 키를 입력하는 공통 API 키 설정 모달을 가진 메인 화면 파일
- `public/styles/main.css`: 메인 화면의 공통 상단 영역, 탭 메뉴, `스크립트` 탭 레이아웃, `이미지` 탭 씬 카드 레이아웃, `영상` 탭 씬 카드 레이아웃, 씬별 이미지 생성 로딩/오류/미리보기 상태, 씬별 영상 생성 로딩/오류/재생 상태, 스크립트 생성 상태, DB 프로젝트 목록 선택 모달, 공통 API 키 설정 모달 스타일 파일
- `public/scripts/main.js`: 탭 전환, 스크립트 설정 입력 보조, 장면 수 스텝 조절, 서버 DB 프로젝트 목록 조회와 선택 프로젝트 고유 ID 기반 복원, 저장된 `project.json`을 기준으로 연결된 스크립트 JSON과 DB에 저장된 이미지/영상 URL 상태를 다시 읽어 복원하는 흐름, `scene-script.json` 호환 불러오기, OpenAI 기반 스크립트 생성 요청, 씬별 이미지 생성 요청과 DB URL 기반 이미지 카드 상태 렌더링, EvoLink.AI Kling 기반 씬별 영상 생성 요청과 DB URL 기반 비디오 카드 상태 렌더링, 프로젝트/씬 단위 오류 문구 표시, 공통 API 키 설정 모달 열기/닫기와 브라우저 저장소 저장을 처리하는 화면 스크립트 파일

## 권장 파일 배치

- 새 화면이나 기능을 추가할 때 파일 생성, 이동, 분리 위치는 해당 책임에 맞게 결정합니다.

## 변경 원칙

- 파일 생성, 저장, 이동 위치는 폴더 책임을 기준으로 결정합니다. 
- 파일을 새로 만들 때 해당 파일 유형에 맞는 폴더가 있는지 먼저 확인합니다. 없으면 폴더를 생성한 뒤 배치합니다.
- 새 폴더, 새 빌드 도구, 새 프레임워크, 대규모 리팩터링은 사용자에게 이유와 범위를 먼저 알리고 승인을 받은 뒤 진행합니다.
- 작업 중인 task 범위를 넘어서는 구조 정리는 별도 요청 없이는 하지 않습니다.
- 새 폴더를 생성하면 `## 현재 프로젝트 폴더 구조와 책임`에 해당 폴더와 역할을 추가합니다.
- 기존 폴더에 현재 구조를 설명하는 데 중요한 파일이 추가되거나 대표 파일 구성이 바뀌면, `## 현재 프로젝트 폴더 구조와 책임`에 해당 폴더의 대표 파일 예시와 각 파일의 한 줄 역할을 실제 구조에 맞게 갱신한다.
