# Harness

## 목적

이 문서는 세션 시작 시 AI 에이전트에게 하네스 엔지니어링 개념을 알리고, 에이전트가 규칙을 벗어났을 때 어떤 문서에 무엇을 추가하거나 수정해야 하는지 판단하는 기준입니다.

## 하네스 엔지니어링 기준

이 저장소에서 하네스 엔지니어링은 모델 성능 자체보다, AI 에이전트가 안정적으로 작업하도록 만드는 작업 환경 설계를 뜻합니다.

- 에이전트는 구현뿐 아니라 현재 작업 흐름이 안전하고 반복 가능하게 유지되는지도 함께 확인합니다.
- 같은 의미의 규칙은 담당 문서에 한 번만 둡니다.

## 하네스 이탈 대응

- 하네스 이탈은 에이전트가 스스로 감지하거나 사용자가 지적하는 두 가지 경우로 발생합니다.
- 사용자가 하네스 이탈을 지적하면, 에이전트는 바로 해명하거나 반박하지 않고 어떤 절차가 지켜지지 않았는지 먼저 인정한 뒤, 아래 절차에 따라 확인하겠다는 점을 사용자에게 먼저 알립니다.

- 에이전트 작업 흐름 이탈이 발생하면 아래 순서로 대응합니다.

1. 어떤 절차가 지켜지지 않았는지 확인합니다.
2. 기존 문서에 이미 해당 규칙이 있는지 확인합니다.
3. 있다면 문서를 늘리지 말고 해당 규칙이 포함된 문서와 `AGENTS.md`의 트리거 조건을 보강합니다.
4. 없다면 어느 문서에 무엇을 추가해야 하는지 사용자에게 제안하고 승인을 받은 뒤 반영합니다.
5. 반영 후 관련 문서끼리 중복되거나 충돌하지 않는지 확인합니다.

## 규칙 추가 판단

새 규칙이나 제약을 추가하기 전에는 아래 순서로 판단합니다.

1. 기존 문서 중 어느 문서가 이 규칙을 담당해야 하는지 확인합니다.
2. 기존 문서에 넣으면 역할이 섞이거나 문서가 비대해지는지 확인합니다.
3. 여러 문서에 걸친 운영 기준이면 독립 문서가 더 적합한지 확인합니다.
4. 새 컨텍스트가 반드시 알아야 하는 기준이면 `AGENTS.md` 또는 먼저 읽을 문서 목록에서 참조되게 합니다.
