# Style Guide

## 목적
이 문서는 AI 에이전트가 코드를 작성할 때 따라야 할 HTML, CSS, JavaScript, 주석 규칙의 기준입니다.

## 공통 원칙

- 처음 보는 사람도 읽을 수 있도록 과도한 난이도의 패턴은 피합니다.

## HTML 규칙

- 시맨틱 태그를 우선 사용합니다.
- 화면 섹션, 카드, 입력 영역, 상태 영역은 구조가 읽히게 마크업합니다.
- 버튼, 입력창, 진행 상태처럼 사용자 상호작용 요소는 역할이 드러나는 이름과 텍스트를 사용합니다.

## CSS 규칙

- 색상, 간격, 반경, 그림자처럼 반복되는 값은 가능한 한 CSS 변수로 정리합니다.
- 화면 작업에서는 목표 레이아웃과의 시각적 유사성을 우선합니다.
- 반응형은 기본적으로 깨지지 않는 수준까지 맞추고, 현재 task 범위 밖의 완전한 다중 디바이스 최적화는 요구될 때만 확장합니다.

## JavaScript 규칙

- 변수와 함수 이름은 역할이 드러나게 작성합니다.
- DOM 탐색은 필요한 범위에서 명시적으로 수행합니다.
- 전역 오염을 줄이고, 화면별 책임이 보이게 코드를 구성합니다.
- 비동기 처리 상태는 로딩, 성공, 실패가 구분되게 작성합니다.
- 파일 저장, API 호출, 폴링, 상태 갱신은 사용자에게 보이는 결과 기준으로 구현합니다.

## 주석 규칙

- 초보자 기준으로 이해를 돕는 주석은 허용합니다.
- 주석은 기본적으로 한글로 작성합니다.
- 왜 이런 분기나 상태 처리가 필요한지처럼 코드만으로 바로 드러나지 않는 맥락을 우선 기록합니다.
- 함수에는 어떤 기능을 하는 함수인지 꼭 주석을 작성합니다.

## 금지 또는 제한 규칙

- 현재 task와 무관한 프레임워크, 라이브러리 도입은 하지 않습니다.
- 보안상 민감한 실제 API 키 값을 코드, 문서, 로그에 적지 않습니다.
- “나중에 쓸 수도 있다”는 이유만으로 미사용 옵션, 더미 API, 가짜 설정 필드를 추가하지 않습니다.
