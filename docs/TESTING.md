# Testing

## 목적

이 문서는 AI 에이전트가 코드 구현 후 검증하는 기준과 절차를 정의합니다.

## 검증 원칙

각 task 구현 후 아래 순서로 검증합니다.

1. 관련 코드 수정 완료 여부 확인
2. 현재 실행 확인 모드에 맞는 정적 점검 또는 로컬 실행 확인
3. `docs/STYLEGUIDE.md` 기준으로 HTML/CSS/JavaScript 규칙 위반 여부 확인
4. 현재 task의 구현 범위와 완료 조건 일치 여부 확인
5. 진행 현황 문서 갱신

## 실행 확인 모드

현재 저장소의 기본 실행 확인 모드는 `수동 실행 확인`입니다. AI가 직접 실행 확인까지 수행하면 토큰 소모가 크기 때문에 기본은 사람이 실행과 결과 확인을 담당합니다.

- `수동 실행 확인`: 에이전트는 코드와 설정을 준비하고, 사람은 실제 실행과 결과 확인을 담당합니다.
- `에이전트 확인`: 필요 시 에이전트가 로컬 실행 명령으로 서버 또는 빌드 동작을 확인합니다.

## 수동 실행 확인 모드

- 에이전트는 코드 눈검토, 관련 파일 점검, 기본 정적 확인, task 문서의 `수동 작업`과 `결과 확인` 절차 정리까지만 수행합니다.
- 사람은 실행 파일 실행, 브라우저 접속, 버튼 동작, 저장 결과, 로딩 상태, 실패 메시지처럼 실제 실행과 결과 확인을 수행합니다.
- 수동 확인 항목은 현재 구현된 범위만 대상으로 삼습니다.
- 아직 연결되지 않은 다음 단계 기능을 현재 task 확인 항목에 섞지 않습니다.
- 실패 시 어떤 화면, 어떤 버튼, 어떤 메시지를 기록해야 하는지 분명해야 합니다.

## 에이전트 확인 모드

- 사용자가 요청하거나 현재 저장소 흐름상 필요할 때만 로컬 실행 확인을 수행합니다.
- 실행 명령은 현재 저장소의 기존 스크립트와 서버 구조를 우선 사용합니다.
